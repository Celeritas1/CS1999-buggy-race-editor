Buggy Creator Subroutine

I believe I have made a viable system editor that allows users to easily navigate through and be able to create a buggy with little issue.

One of the first main features is the buggy creator subroutine itself. With it, there are many features that provide the user easy accessiblity fill out required details with little effort.

One such example would be the dropdown menus that does not require the user to enter the specific power_type or attack option themeselves. This could risk spelling errors that simply prolong the form being submitted

Before a buggy submission is committed to the database, there are several backgroumd subroutines running. One is the validation function which detemrines if the buggy options selected satisfy the game rules and if so, it will then be passed onto the calculation function. This determines the cost of the options chosen and will then finally commit to the desired database using SQL Query.

Another function I have is the Auto-Filling of empty input fields. My system has a preset in place to fill in all fields if necessary and for tests to be run with quick efficiency. This also acts as a way for users to easily make a buggy with a click of a button

Another function is my random buggy generator which in a way acts as another Auto-filing function that allows the user to easily create a buggy with random options with ease. I believe the phase of testing and accessbility has been achieved with this idea quite well

Buggy Records

Another function is that my editor allows the user to review their last buggy save made. This is useful incase a developer or client would like to see their past buggies saved as well as simply view their buggy currently saved in the database.

Furthermore, this acts as a great testing mechanism for whenever a change is made to the buggy creating func, you can view if the details have successfully saved or not as it will past on this page.
